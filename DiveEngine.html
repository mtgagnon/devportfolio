Dive Game Engine

Overview

Dive engine is a project I started for the Game Engine Architecture Class at the University of Michigan. We went through the process of creating first a game, and then slowly gutting it and adding functionality week by week until it became a full fledged 2D game engine. This engine utilizes Lua and LuaBridge for scripting, SDL for 2D rendering as well as input, Box2D for the 2D physics and collision manager, GLM for math, and Rapidjson for parsing scene files.

Controller Support

After creating the engine I wanted to add new ways of interacting with my game engine. Something pretty simple to start was to add controller support using SDL. I was really excited about this although, I found that with Nintendo Switch Controllers stick drift was a big problem unfortunately.

Plans

After spending a half year on this project I am not ready to let this go. During the summer I plan to add UI elements like a scene editor and also a really big one I plan on doing is add in 3D support using OpenGL.
